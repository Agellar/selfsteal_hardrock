# Создание selfsteal-сайтов под новый музыкальный жанр

Пошаговый playbook для клонирования этого репозитория в новый
сайт-радиостанцию другого жанра (джаз, техно, synthwave, lo-fi, классика и т.д.),
который служит **selfsteal-маскировкой** (decoy-сайт для xray Reality / VLESS).

Документ собран на основе всех фиксов и решений, накопленных в
`hardrock.legendaryfm.uk`. Идите по разделам сверху вниз — в конце есть
финальный чек-лист перед публикацией.

---

## 0. Что такое selfsteal и какие требования он диктует

В Reality сервер xray на `:443` форвардит **весь не-Reality трафик** (обычные
браузеры, активные проберы цензора) на свой `target` — локальный веб-сервер,
который отдаёт **настоящий сайт с валидным TLS-сертификатом**. Если этот decoy
выглядит как живой, уникальный сайт — маскировка работает. Если он палится
(битый шаблон, внешние зависимости, дефолтная заглушка, ошибка сертификата) —
маскировка проваливается.

Отсюда **жёсткие требования к decoy**, которым обязан удовлетворять каждый сайт:

| Требование | Почему |
|---|---|
| **Ноль внешних запросов при загрузке** | Любой запрос к Google Fonts / CDN / аналитике выдаёт типовой шаблон и создаёт точку отказа. Всё — со своего origin. |
| **Валидный публичный TLS-сертификат** на decoy | Браузер/пробер, попавший на сайт, не должен увидеть ошибку сертификата. |
| **Уникальность каждого деплоя** | Одинаковый контент на N серверах фингерпринтится по хешу. Меняйте тексты, палитру, имена. |
| **Правдоподобная «живость»** | sitemap, robots, JSON-LD, canonical, рабочие внутренние ссылки, актуальный год. |
| **Согласованность privacy-политики с реальностью** | Заявляете «не собираем данные» — значит логи без IP, никаких трекеров. |
| **Рабочий контент из целевого региона** | Радиопотоки должны играть из СНГ без VPN. |

## 1. Архитектура репозитория (что копируем)

```
.
├── index.html          # Главная: плеер, сетка станций, тизеры, факты
├── history.html        # Контентная статья №1 (история жанра)
├── artists.html        # Контентная статья №2 (исполнители)
├── subgenres.html      # Контентная статья №3 (поджанры/стили)
├── guide.html          # Контентная статья №4 (как слушать)
├── about.html          # О проекте + Политика конфиденциальности + Cookie
├── 404.html            # Стилизованная страница ошибки (реализм декоя)
├── favicon.ico         # Бинарный favicon (реальные сайты его отдают)
├── robots.txt          # Разрешение индексации + ссылка на sitemap
├── sitemap.xml         # Карта 6 страниц
├── assets/
│   ├── styles.css      # Все стили (общие, через CSS-переменные)
│   ├── app.js          # Плеер, массив STATIONS, тема, меню, cookie-баннер
│   ├── fonts.css       # @font-face → локальные woff2
│   ├── fonts/          # Самохостящиеся шрифты (.woff2)
│   └── logo.svg        # Логотип (для JSON-LD и иконок)
├── deploy/
│   └── install.sh      # Интерактивный установщик: Caddy в Docker (/opt/caddy),
│                       # реальный TLS на 127.0.0.1:9443 (Reality target)
├── .github/workflows/
│   └── deploy.yml      # Автодеплой по push (rsync + install.sh --sync-only)
└── docs/               # Этот гайд
```

**Ключевой принцип:** общий «каркас» (шапка, плеер, подвал) дублируется в каждом
HTML-файле осознанно — сайт остаётся чисто статическим, без шаблонизатора. Логика
и стили вынесены в `assets/` и не дублируются.

## 3. Слой 1 — Бренд и идентичность

Замените по всем файлам:

- **Название** `Iron Wave` / `Iron Wave Radio` → ваш бренд.
- **Подпись** `Hard Rock Radio` → `<Genre> Radio`.
- **Логотип** `assets/logo.svg` — перерисуйте SVG (форма + цвет). Заодно поменяйте
  inline-favicon (`<link rel="icon" ... data:image/svg+xml ...>`) в `<head>`
  каждой страницы и бинарный `favicon.ico` в корне.
- **Страница `404.html`** — поменяйте бренд/тексты/цвет под жанр (её отдаёт Caddy
  при ненайденном пути — должна выглядеть как часть сайта).
- **Контактный e-mail** во всех местах (`mailto:`, JSON-LD, политика).

> ⚠️ **Анти-фингерпринт:** не оставляйте дефолтных строк. Меняйте не только имя,
> но и формулировки заголовков/слоганов, порядок секций, набор emoji. Цель — чтобы
> два разных decoy не совпадали по содержимому побайтово.

## 4. Слой 2 — Контент жанра

Контент жив в двух местах:

1. **Статические HTML** контентных страниц (`history`, `artists`, `subgenres`,
   `guide`) — перепишите тексты под жанр. Структура (TOC, `.prose`, `.callout`,
   `.info-grid`, `.profile`) переиспользуется как есть.
2. **Тизеры на `index.html`** (таймлайн, сетка артистов, факты) — короткие версии,
   ведущие на полные страницы.

Адаптация по разделам:

| Файл | Хард-рок | Чем заменить для нового жанра |
|---|---|---|
| `history.html` | Эволюция от блюз-рока до ренессанса | Хронология вашего жанра |
| `artists.html` | Led Zeppelin, AC/DC… | Ключевые исполнители жанра |
| `subgenres.html` | Heavy, glam, grunge… | Поджанры/стили |
| `guide.html` | Как слушать интернет-радио | Можно оставить почти как есть (универсально) |
| `about.html` | О проекте + политики | Поменять только бренд/жанр, политики — см. §8 |

`guide.html` про «как работает интернет-радио / доступ из СНГ / битрейт» —
жанро-независим, правьте минимально.

## 5. Слой 3 — Радиопотоки (критично для «живости»)

Потоки задаются массивом `STATIONS` в начале `assets/app.js`. Каждый поток
**обязательно** проверяйте на: (а) HTTPS, (б) реальный `audio/mpeg`/`audio/aac`,
(в) доступность из целевого региона.

### 5.1 Тест доступности потоков

```bash
test_stream() {
  url="$1"; name="$2"
  code=$(curl -sSL -m 10 -o /dev/null \
    -w "%{http_code}|%{content_type}|%{url_effective}" -r 0-1 "$url" 2>/dev/null)
  printf "%-28s -> %s\n" "$name" "$code"
}
test_stream "https://stream.laut.fm/jazz" "laut.fm jazz"
test_stream "https://listen.181fm.com/181-classichits_128k.mp3" "181 classic"
```

Берём поток, только если код `200`/`206` и `content_type` начинается с `audio/`.
Редирект `302` — нормально, если по `url_effective` итог `audio/*` (так делает
Rock Antenne). `404`/`000` — отбрасываем.

### 5.2 Где брать рабочие потоки

- **laut.fm** — `https://stream.laut.fm/<имя>` — огромный каталог по жанрам (DE).
- **181.fm** — `https://listen.181fm.com/<канал>_128k.mp3` (US).
- **0N Radio / radionetz** — `https://0n-<канал>.radionetz.de/0n-<канал>.mp3` (DE).
- **Rock Antenne / профильные сети** — у каждой свои поджанровые потоки.
- Публичные каталоги: радио-агрегаторы и `radio-browser.info` (берите прямой
  stream-URL, перепроверяйте на HTTPS).

> Источники из **Германии и США** обычно доступны из СНГ без VPN и без
> геоблокировки — это проверенный выбор. Избегайте потоков, которые редиректят на
> `http://` (mixed content заблокируется на HTTPS-странице).

### 5.3 Формат элемента STATIONS

```js
{ name: "Blue Note Live", emoji: "🎷", country: "🇩🇪", bitrate: "MP3 128k",
  description: "Краткое описание станции.",
  genres: ["Jazz", "Bebop", "Swing"],
  url: "https://stream.laut.fm/jazz", isLive: true }
```

Плеер уже умеет: переключение по карточке/селекту, **watchdog 12 c** на зависший
поток и **автопереход на следующую станцию** при ошибке — менять логику не нужно,
только данные.

## 6. Слой 4 — Шрифты (self-host) и палитра

### 6.1 Самохостинг шрифтов (обязательно — убирает внешний вызов)

```bash
mkdir -p assets/fonts
UA="Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 Chrome/120 Safari/537.36"
# Запросить CSS под нужные начертания (пример: Inter + нужные веса)
curl -sS -A "$UA" \
  "https://fonts.googleapis.com/css2?family=Inter:wght@400;600;800&display=swap" \
  -o assets/fonts.css
# Скачать все woff2 и переписать ссылки на локальные
for url in $(grep -oE "https://[^ )]+\.woff2" assets/fonts.css | sort -u); do
  f=$(basename "$url"); curl -sS -A "$UA" "$url" -o "assets/fonts/$f"
  sed -i "s#$url#fonts/$f#g" assets/fonts.css
done
# Проверка: не должно остаться gstatic-ссылок
grep -c gstatic assets/fonts.css   # => 0
```

> Для русскоязычного сайта убедитесь, что в наборе есть **кириллические сабсеты**
> (Google отдаёт несколько `unicode-range` блоков — качаются все, не удаляйте).
> Затем обновите `--font-display` / `--font-body` в `assets/styles.css`.

### 6.2 Палитра

Вся палитра — в CSS-переменных в начале `assets/styles.css` (`:root` и
`[data-theme="light"]`). Поменяйте `--color-primary`, `--color-bg`, surface-цвета
под настроение жанра (джаз — тёплый янтарь/индиго; техно — неон/графит; lo-fi —
пастель). Светлая/тёмная темы переключаются автоматически.

## 7. Слой 5 — Мета, разметка, карты сайта

Подставьте `DOMAIN` и контент во **все** места (по одному `sed` на домен):

```bash
DOMAIN="https://jazz.example.com"
sed -i "s#https://hardrock.legendaryfm.uk#$DOMAIN#g" robots.txt sitemap.xml *.html
```

Проверьте и обновите вручную:

- `<title>` и `<meta name="description">` — под жанр, на каждой странице уникально.
- `<link rel="canonical">` — абсолютный URL страницы (после sed уже верный хост).
- `og:title` / `og:description` / `og:url` на `index.html`.
- **JSON-LD** (`<script type="application/ld+json">`): `RadioStation`/`Organization`
  + `BroadcastService` + `WebSite` на главной; `Article` + `BreadcrumbList` на
  статьях; `AboutPage` на about. Поменяйте `name`, `genre`, `headline`, `email`.
- `sitemap.xml` — обновите `<lastmod>` на дату публикации.
- **Динамический год** в подвале уже автоматический (`<span data-year>` +
  `app.js`) — трогать не нужно.

Валидация перед коммитом:

```bash
# JSON-LD во всех страницах — валидный JSON
python3 - <<'PY'
import re,json,glob
for f in sorted(glob.glob("*.html")):
  for b in re.findall(r'application/ld\+json">(.*?)</script>', open(f).read(), re.S):
    json.loads(b); print(f, "ok")
PY
# sitemap — валидный XML
python3 -c "import xml.dom.minidom; xml.dom.minidom.parse('sitemap.xml'); print('sitemap ok')"
# не осталось внешних вызовов
grep -rn "googleapis\|gstatic\|google-analytics\|metrika\|mc.yandex" --include=*.html --include=*.css --include=*.js . || echo "external refs: none"
```

## 8. Слой 6 — Соответствие 152-ФЗ и честность политики

Политика конфиденциальности и cookie живут в `about.html` (`#privacy`,
`#cookies`). Правила, чтобы заявления были **достоверны**:

- Сайт не должен иметь форм ввода ПДн, аналитики, рекламы, сторонних трекеров.
- Технические настройки (`theme_preference`, `cookie_consent`) хранятся в
  `localStorage` — и в тексте политики указан именно `localStorage`
  (не `sessionStorage`!). Код в `app.js` уже использует `localStorage` —
  **не рассинхронизируйте текст и код**.
- Укажите **реальный контакт оператора** (e-mail). Без контакта политика «пустая».
- На сервере **отключите/анонимизируйте access-логи** (см. §9) — иначе фраза «не
  собираем данные» становится ложью, и это юридический и маскировочный риск.

## 9. Деплой (Docker + Caddy за xray Reality)

### 9.1 Принцип

Decoy слушает `127.0.0.1:<port>` (по умолчанию `9443`) с **настоящим TLS**; xray
Reality на `:443` указывает на него: `dest/target = "127.0.0.1:9443"`,
`serverNames: ["<ваш-домен>"]`. Деплой ставит **Caddy в Docker** (`/opt/caddy`,
`docker compose`) — рядом с remnanode, без native-caddy/systemd.

### 9.2 Установщик `deploy/install.sh`

Один интерактивный скрипт делает всё: спрашивает домен, порт и e-mail для ACME,
рендерит `docker-compose.yml` + `Caddyfile`, поднимает контейнер, получает
сертификат Let's Encrypt и заливает сайт в `/opt/caddy/html`.

```bash
git clone <repo-url> && cd <site>
sudo bash deploy/install.sh
```

Флаги и неинтерактивный режим:

```bash
sudo bash deploy/install.sh --dry-run     # сгенерировать+проверить конфиг, ничего не менять
sudo bash deploy/install.sh --sync-only   # обновить только файлы сайта (для CI)
sudo DOMAIN=example.com SELF_STEAL_PORT=9443 ACME_EMAIL=you@example.com \
     bash deploy/install.sh --yes         # без вопросов
```

**Предусловие:** на сервере уже работает **remnanode** (Docker), DNS домена
указывает на сервер, а Reality-инбаунд (serverName + dest=`127.0.0.1:9443`)
настроен в панели — установщик его не трогает.

Что внутри хардненинга (реализм декоя, переносится как есть):
- настоящий TLS + `proxy_protocol` приёма от xray;
- **без SPA-catch-all**, отдельная стилизованная `404.html`;
- кэш-заголовки (ассеты — `immutable`, HTML — короткий) и security-заголовки;
- сжатие zstd/gzip; логи **без IP** (согласовано с privacy-политикой).

### 9.3 Сертификат

Для Let's Encrypt нужен **открытый TCP-порт 80** (HTTP-01) — TLS-ALPN на 443
невозможен, его держит xray. Установщик откроет 80 в `ufw`; в облачной
security-group откройте порт вручную. Порт 443 не трогается (его держит xray),
9443 — только loopback.

### 9.4 Проверка

```bash
# реальный путь: 443 -> xray Reality -> selfsteal fallback
curl -skI --resolve <домен>:443:127.0.0.1 https://<домен>/ | head -n1
# сертификат должен быть Let's Encrypt, не self-signed:
echo | openssl s_client -connect 127.0.0.1:9443 -servername <домен> 2>/dev/null \
  | openssl x509 -noout -issuer
```

> Если поверх накатывали официальный установщик selfsteal — он перезапишет
> `/opt/caddy/Caddyfile` и вернёт SPA-catch-all. После такого reinstall просто
> запустите `deploy/install.sh` снова — он восстановит хардненинг.

### 9.5 Автодеплой (опционально)

`.github/workflows/deploy.yml` выкатывает по push в `main`: rsync репозитория на
сервер + `install.sh --sync-only` (обновляются **только файлы сайта**, конфиг,
порты и сертификат не трогаются). Секреты в GitHub: `SSH_HOST`, `SSH_USER`,
`SSH_KEY` (приватный **выделенный** deploy-ключ), опц. `SSH_PORT`. Полную
установку (`sudo bash deploy/install.sh`) один раз выполните вручную. Приватный
ключ — только в GitHub Secrets, никогда в коде/чате.

- [ ] Можно изменить порядок/состав секций на главной.
- [ ] Уникальные `<title>`/`description`/JSON-LD.

## 12. Список багов донора, которые уже исправлены (не вносите заново)

Эти дефекты были в исходном шаблоне и **уже устранены** — при ручной правке не
верните их обратно:

1. **Битый HTML** `<li><li>…</li></li>` в подвале — двойная вложенность.
2. **Рассинхрон станций** — кнопки плеера были захардкожены отдельно от данных;
   теперь всё рендерится из массива `STATIONS`.
3. **Тема не сохранялась**, хотя политика обещала хранение — теперь пишется в
   `localStorage`, и текст политики приведён в соответствие.
4. **Внешний Google Fonts** — заменён на self-host.
5. **Нерабочие/устаревшие потоки** (например, 404) — заменены проверенными.
6. **Протухший `© 2025`** — заменён динамическим годом.
7. **Отсутствие фоллбэка потока** — добавлены watchdog и автопереключение.
